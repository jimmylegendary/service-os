# Today It — Loops

## One-line flow
Mental clutter -> intentional acceptance into a trusted today surface -> action -> completion or controlled reappearance

## Main product loop
Capture -> Review/Accept -> Today -> Done / Snooze / Waiting -> Resurface

## Sub-loop: acceptance
Input:
- captured item
- incoming request
- resurfaced unfinished item

Function:
- decide whether this belongs on the trusted execution surface for now

Output:
- accepted_today
- snoozed
- waiting
- remains in capture/inbox

## Sub-loop: resurfacing
Input:
- unfinished or snoozed item
- minimal timing/context rules

Function:
- determine whether the item should become visible again

Output:
- visible again in capture/inbox or today preview
- still hidden until a later moment

## User loop
1. unload mental clutter quickly
2. accept only what belongs in today
3. act from a smaller trusted surface
4. close the loop through done, snooze, or waiting
5. trust the system to bring relevant things back

## Anti-loop warning
The product fails if the user repeatedly re-enters a backlog-management loop:
- browse too much
- organize too much
- schedule too much
- review too much
- think too much before acting
