# Fractal Method

## Core idea
Every meaningful problem can be viewed as:

`input -> function -> output`

Then each function can be decomposed into a smaller flow with the same structure.

This applies to:
- service strategy
- user flows
- engineering systems
- automation pipelines
- collaboration between humans and agents

## Use this at three levels

### Level 1. System purpose
Example:
- input: human mental clutter
- function: reduce to a clear present-action set
- output: actionable today surface

### Level 2. Main flow
Example:
- capture
- classify
- reduce
- prioritize
- present
- complete / defer / re-surface

### Level 3. Sub-flow
Example for `prioritize`:
- input: candidate tasks + context
- function: relevance / urgency / energy / dependency filtering
- output: now / next / later-today set

## Rule
For any important service or feature, be able to explain:
1. the one-line flow
2. the main loop
3. at least one decomposed sub-loop

If not, the design is probably still too vague.
