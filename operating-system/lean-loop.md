# Lean Loop

## Why this exists
To avoid two failure modes:
- oversized waterfall planning
- directionless feature hacking

## Standard loop

1. define enduring philosophy
2. define current hypothesis
3. define smallest meaningful MVP slice
4. build that slice
5. observe reality
6. record learning
7. update roadmap and next slice

## Distinguish these carefully

### Enduring philosophy
Hard to change later:
- what problem matters
- what the product should optimize for
- what it should never become

### Current hypothesis
Expected to change:
- target segment guess
- onboarding assumptions
- feature packaging
- pricing assumptions

### Current MVP
What gets built now.
This must be aggressively narrow.

### Learning record
What reality taught us.
This should change the next step.

## Rule of thumb
Think broadly, build narrowly, revise honestly.
