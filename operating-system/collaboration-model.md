# Collaboration Model

## Goal
Describe the higher-level abstraction behind Jimmy + OpenClaw + coding-agent collaboration.

`WORKFLOW.md` is the practical operating protocol.
This file explains the conceptual model underneath it.

## Conceptual pipeline

Direction
-> synthesis
-> bounded brief
-> implementation
-> self review
-> human judgment
-> learning capture
-> reusable pattern extraction

## Design assumptions

- Jimmy is not expected to manually implement every feature.
- OpenClaw preserves coherence and translates philosophy into execution.
- coding agents are focused execution engines, not product owners.
- durable learning matters because recurring patterns should eventually become reusable skills.

## What OpenClaw must protect

1. philosophy coherence
2. MVP boundary discipline
3. document/code alignment
4. bounded implementation packets
5. learning capture for future reuse

## Skill extraction lens

When collaboration works repeatedly, ask:
- what triggered this workflow?
- what sequence of steps kept it effective?
- where did Jimmy provide high-value intervention?
- what parts can be generalized?
- can this become a reusable skill or playbook?
